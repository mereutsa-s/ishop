Admin panel

Dashboard

Sales report by month, order statistics, top selling products, categories, brands and many other reports from the dashboard.

Product

Categories For categories image, title and status is required.

SubCategories For sub categories image, title, status and the category is required. Also admin can set if the sub category will be in the featured list.

Brands For brands image, title, status is required. Also admin can set if the brand will be in the featured list.

Attributes Attribute is being used in the product. Admin can add attribute and the values of the attribute from here. While adding a product attribute can be attached.

Tax Rules Tax amount in flat rate or percent is acceptable. While adding a product tax rules can be attached.

Shipping Sales Admin can add shipping rules for different location for a product. Also admin can add pickup point shipping price different location. All the location is being found in the frontend > resources > countries.json file. While adding a product shipping rules can be set.

Collections Collection of products is set from here by adding title. Admin can hide any collection by setting the status private. A product can be added in different collection. These collection will be shown in the homepage.

Bundle Deals Admin can add bundle deals from here. If admin want to give the user a free item if the user buy a minimum quantity of that product, this option can be found here.

Vouchers Voucher option can be added from here.

Products All the necessary data have to added here of a product. If the product have different attributes, admin have to add from the Product Inventory section. Also, admin can set the inventory here by different attributes.

Flash Sales

Admin can set flash sale option from here, by adding start and end time. While adding a product in the flash sale, an offered price and quantity(How much item can be sold) have to set. Admin can make any flash sale private at any time by setting he status private.

Orders

All the placed orders can be view from here. Admin can change the status form the detail page of an order. Also, PDF can be generated form the bottom of the detail page.

Users can cancel an order by giving a message, until the status is pending.

Admin can refund a cancelled order.

Rating reviews

Product reviews can be found here.

Users

All the users can be view from here. Admin can delete any user form here.

Subscription

SubscribersUsers, who subscribed from the frontend website, can be found here.

Send emailAdmin can send email to the subscribers from the admin panel from here.

Email formatsAsmin can format the emnail template from here, which will be sent to the subscribers.

Roles & permissions

Super admin can add/edit any permission from here. Admin/vendor is unable to edit his own permission.

Admins/vendors

Admin can add new vendor or other admin user from here.

Withdrawal

This section is for vendors.

RequestsVendors can request a withdrawal from here. And admin can approve/cancel/delete that request.

Currently, minimum amount for withdrawal is set to 50. You can change it from backend > config > constants file from withdraw > MIN_AMOUNT. Also, you have to change from admin > mixin > util file from withdraw > minAmount

AccountsVendor bank account information(Where to send withdrawal money) found here.

UI

PagesContent of the static pages can be added from here.

Home SliderThe setting of the top section of the home page can be found here. Admin have to upload the images from here. In the edit section of each image defines the source of the products after clicking of that image. There is several options for the sources. You can set the source by categories, sub categories, brands and tags.

BannerAll the banners will be found here.

FooterFooter links can be added from here. There is four columns in the footer section. Each item of columns can be edited from here.

Settings

Currency, store info and payment configuration can be set from here. Store info have to be added in order to add the information in the email send from the server to the users.
